<td><h2 id="owasp_juice_shop_tool_project">OWASP Juice Shop Tool Project</h2>
<p><em>The most trustworthy online shop out there.</em> (<a href="https://twitter.com/dschadow/status/706781693504589824">dschadow</a>) â€” <em>The best juice shop on the whole internet!</em> (<a href="https://twitter.com/shehackspurple/status/907335357775085568">shehackspurple</a>) â€” <em>Actually the most bug-free vulnerable application in existence!</em> (<a href="https://youtu.be/TXAztSpYpvE?t=26m35s">vanderaj</a>) â€” <em>First you</em> ðŸ˜‚ðŸ˜‚<em>then you</em> ðŸ˜¢ (<a href="https://twitter.com/kramse/status/1073168529405472768">kramse</a>)</p>
<p>OWASP Juice Shop is probably the most modern and sophisticated insecure web application! It can be used in security trainings, awareness demos, CTFs and as a guinea pig for security tools! Juice Shop encompasses vulnerabilities from the entire <a href="OWASP_Top_Ten" title="wikilink">OWASP Top Ten</a> along with many other security flaws found in real-world applications!</p>
<h2 id="description">Description</h2>
<figure>
<img src="JuiceShop_Logo.png" title="JuiceShop_Logo.png" alt="JuiceShop_Logo.png" width="200" /><figcaption>JuiceShop_Logo.png</figcaption>
</figure>
<p>Juice Shop is written in Node.js, Express and Angular. It was the first application written entirely in JavaScript listed in the <a href="OWASP_Vulnerable_Web_Applications_Directory_Project" title="wikilink">OWASP VWA Directory</a>.</p>
<p>The application contains a vast number of hacking challenges of varying difficulty where the user is supposed to exploit the underlying vulnerabilities. The hacking progress is tracked on a score board. Finding this score board is actually one of the (easy) challenges!</p>
<p>Apart from the hacker and awareness training use case, pentesting proxies or security scanners can use Juice Shop as a "guinea pig"-application to check how well their tools cope with JavaScript-heavy application frontends and REST APIs.</p>
<p><em>Translating "dump" or "useless outfit" into German yields "Saftladen" which can be reverse-translated word by word into "juice shop". Hence the project name. That the initials "JS" match with those of "JavaScript" was purely coincidental!</em></p>
<h2 id="main_selling_points">Main Selling Points</h2>
<ul>
<li>Free and Open source: Licensed under the <a href="https://github.com/bkimminich/juice-shop/blob/master/LICENSE">MIT license</a> with no hidden costs or caveats</li>
<li><a href="https://github.com/bkimminich/juice-shop#setup">Easy-to-install</a>: Choose between <a href="http://nodejs.org">node.js</a>, <a href="https://www.docker.com">Docker</a> and <a href="https://www.vagrantup.com/downloads.html">Vagrant</a> to run on Windows/Mac/Linux</li>
<li>Self-contained: Additional dependencies are pre-packaged or will be resolved and downloaded automatically</li>
<li>Self-healing: The simple SQLite and MarsDB databases are wiped and repopulated from scratch on every server startup</li>
<li>Gamification: The application notifies you on solved challenges and keeps track of successfully exploited vulnerabilities on a Score Board</li>
<li>Re-branding: Fully customizable in business context and look &amp; feel to your own corporate or customer requirements</li>
<li>CTF-support: Challenge notifications optionally contain a flag code for your own <a href="https://github.com/bkimminich/juice-shop-ctf">Capture-The-Flag events</a></li>
</ul>

# Road Map and Getting Involved

Juice Shop is already implemented, properly tested and [has been
promoted](https://github.com/bkimminich/juice-shop/blob/master/REFERENCES.md#web-links)
and [demonstrated or live-hacked on various occasions including OWASP
events](https://github.com/bkimminich/juice-shop/blob/master/REFERENCES.md#conference-and-meetup-appearances).

security trainings as well as [in university lectures or published

## Getting Involved

Involvement in the development and promotion of OWASP Juice Shop is
actively encouraged\! You do not have to be a security expert or a
programmer to contribute. Some of the ways you can help are as follows:

  - use Juice Shop in your own hacker or awareness trainings
  - use Juice Shop as a "guinea pig" for your security tools
  - provide ideas for new vulnerabilities and challenges
  - provide feedback via [email](mailto:bjoern.kimminich@owasp.org),
    [chat](https://gitter.im/bkimminich/juice-shop) or by [opening an
    issue](https://github.com/bkimminich/juice-shop/issues)
  - help translating the user interface on
    [Crowdin](https://crowdin.com/project/owasp-juice-shop)
