<td><h2 id="owasp_financial_information_exchange_security">OWASP Financial Information Exchange Security</h2>
<p>OWASP Financial Information Exchange Security is a project aimed at raising awareness of security when implementing, developing or working with the FIX protocol. The project aims to create guidelines for implementors, providing them with best practice guidance relating the deploying FIX, guidelines for security professionals when penetration testing FIX, and also a number of FIX related security tools written in Java.</p>

<h2 id="what_is_financial_information_exchange_security">What is Financial Information Exchange Security?</h2>
<p>OWASP Financial Information Exchange Security provides:</p>
<ul>
<li>Guidelines for "implementors" of the FIX protocol relating to security best practice. The guidelines will aim to thwart the common vulnerabilities observed in a typical FIX deployment.</li>
<li>Guidelines for security professionals relating to how they can perform better security assessments of FIX endpoints. At the moment this is expected to be in the form of an "attack cheat-sheet"</li>
<li>FIX Fuzzer - Java application to fuzz implementations of the FIX protocol, identifying common application vulnerabilities and business logic attacks</li>
</ul></td>

# Road Map and Getting Involved

## Road Map

  - Gather interested community members

<!-- end list -->

  - Develop check-list for penetration testing FIX (Financial
    Information Exchange) endpoints

<!-- end list -->

  - Develop best practice guidelines when implementing the FIX protocol
    (from a security perspective)

<!-- end list -->

  - Develop Java based FIX client to use when performing security review
    of FIX endpoint (kind of like ZapProxy but for FIX)

## Getting Involved

As of January 2014, the priorities are:

  - Start work on the Java FIX Security tool
  - Start work on the FIX Security best practice document

Involvement in the development and promotion of OWASP FIX Security is
actively encouraged\! You do not have to be a security expert in order
to contribute. Some of the ways you can help:

  - FIX architects, let us know your security concerns and problems you
    face when deploying new FIX architecture.

  - Java developers, one of the projects deliverables is a Java FIX
    Security tool, any development help would be great\!
